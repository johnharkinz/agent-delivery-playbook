## Rules

- Be opinionated. When multiple words exist for the same concept, pick the best one and list the others under `_Avoid_`.
- Keep definitions tight. Define what the concept is, not what it does.
- Include only terms specific to this project's domain language.
- Exclude general programming concepts unless they have a project-specific domain meaning.
- Group terms under subheadings when natural clusters emerge. If all terms belong to one cohesive area, a flat list is fine.
- Update this file when a term is clarified, renamed, split, merged, or deliberately rejected.
